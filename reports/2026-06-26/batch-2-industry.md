### [21] AWS SageMaker + NVIDIA Blackwell B200：8 倍训练吞吐提升，NVLink 5 加持
- **来源** ｜ AWS ML Blog ｜ 2026-06-25 ｜ **质量分** ｜ 7/10 ｜ **链接** ｜ https://aws.amazon.com/blogs/machine-learning/optimize-model-training-on-amazon-sagemaker-ai-with-nvidia-blackwell/

AWS 详解 SageMaker 搭配 NVIDIA Blackwell（B200/B300）的训练优化方法。B200 提供 180GB HBM 显存（B300 为 268GB），NVLink 5 互联带宽达 1.8 TB/s 双向，Tensor Core 支持 FP8/MXFP8/NVFP4。实测：1B 参数模型在 8K 上下文长度下开启激活检查点（Activation Checkpointing）后，吞吐提升约 8 倍（约 5.1 万 token/s）。大模型（14B+）因内存带宽瓶颈收益更高。P6-B200 实例已通过 SageMaker Flexible Training Plans 提供预留。

📌 **这意味着**：AI 训练预算较大的企业可通过 FP8 混合精度 + 激活检查点组合显著降本；H100 向 B200 升级的 ROI 节点正在成熟，采购方可启动 TCO 评估。

### [23] AWS 自建 AI Health 分析：MCP 协议 + Bedrock Agent，自然语言查询 AWS 健康事件
- **来源** ｜ AWS ML Blog ｜ 2026-06-25 ｜ **质量分** ｜ 6/10 ｜ **链接** ｜ https://aws.amazon.com/blogs/machine-learning/build-self-service-aws-health-analytics-to-find-actionable-health-insights-with-ai-agents-powered-by-amazon-bedrock/

AWS 开源方案"Chaplin"：通过 Model Context Protocol (MCP) 让 AI Agent 以自然语言查询 AWS Health 数据，提供自助式健康事件分析。架构：MCP 服务器连接 AWS Health API → Bedrock Agent 理解查询意图 → 生成行动建议。目标用户是 DevOps 和 SRE 团队，将 AWS 服务中断、维护事件的分析时间从小时级压缩至分钟级。
